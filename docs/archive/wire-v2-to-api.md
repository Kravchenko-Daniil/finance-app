# Задача: подключить web/v2.html к реальному API

## Контекст
`web/v2.html` — новый единый экран (дизайн Editorial+motion из Claude Design), уже
собран как самодостаточный vanilla-файл (no build/React). Сейчас работает на
**данных-заглушках** из макета (моковые счета ₽/$/€, хардкод дней, запись только
показывает статус). Старые 4 страницы (`index/balances/events/expenses.html`) НЕ
трогаем — v2 лежит рядом.

**Цель:** заменить заглушки на реальные вызовы `/api/*`, чтобы v2.html стал
полностью рабочим. **Визуал и анимации не менять** — только проводка данных.

## Что переиспользовать (логика уже написана в старых файлах)
- `web/app.js` — запись траты + **оффлайн-очередь** (localStorage, flush на online/focus/30s).
- `web/balances.js` — чтение балансов.
- `web/events.js` — POST события (income/expense/exchange/transfer) + undo.
- `web/expenses.js` — дневной лог со стрелками.
- `web/config.js` — `window.AppConfig` (часовой пояс, `today()`, `zonedNoonISO()`), токен.

Токен: `localStorage['appToken']`, заголовок `Authorization: Bearer <token>`.
Все запросы — **относительные** пути (same-origin), CORS не нужен.

## Проводка по экранам (эндпойнты — см. CLAUDE.md, раздел про API)
- **Запись** → `POST /api/expense` `{text}`. Успех → зелёная галочка (уже есть),
  ошибка/оффлайн → в очередь как в app.js. Парсер суммы — на бэке, не дублировать.
- **Балансы** → `GET /api/balances`. Ответ даёт `accounts` (с валютой),
  `primary`/`primary_currency` (главный счёт — крупной цифрой в Tiffany),
  `updated_at`, `timezone`. Убрать моковый массив `accounts`, брать отсюда.
  Валюты — реальные (THB/USDT/RUB/VND), не хардкодить ₽/$/€.
- **День** → `GET /api/day?date=YYYY-MM-DD`. Стрелки листают дни (offset уже есть);
  дату считать по активной зоне через `AppConfig`. «Итого» — из `totals`.
- **События** → `POST /api/event` `{type,from,to,amount,amount_to?,note?,at?}`.
  Счета для сеток from/to — из `GET /api/balances`. Backdate `at` — через
  `AppConfig.zonedNoonISO(date)`. После записи — тост (есть) + **undo** →
  `DELETE /api/event/last`.
- **Настройки** → часовой пояс через `GET/PUT /api/config` (источник правды — KV,
  дропдаун уже есть; список зон согласовать с текущими страницами). Токен —
  сохранять в `localStorage['appToken']`.

## Готчи
- Счёт из макета `main/cash/tinkoff/...` — выкинуть, id счетов приходят из API.
- Не ломать анимации: данные обновлять точечно, `#screen` не пересоздавать зря
  (screenIn не должен моргать на каждом обновлении).
- SW (`web/sw.js`) — при желании добавить `./v2.html` в precache-список `FILES`
  и бампнуть `CACHE`-версию (иначе только рантайм-кэш; для прода норм и так).

## Definition of done
- v2.html полностью работает против реального API (запись, балансы, день, события,
  undo, настройки), оффлайн-очередь трат жива.
- Старые 4 страницы не сломаны.
- Проверить: `curl` smoke на `/api/balances` и запись тест-траты (примеры в CLAUDE.md).
