# docs/ — вся документация проекта

Единое место для всех документов. Код — в `api/`, `web/`, `scripts/`; всё, что
про проект словами (планы, инструкции, история, заметки) — здесь.

## Что в какой папке

| Путь | Что лежит | Зачем |
|------|-----------|-------|
| `CHANGELOG.md` | История изменений по сессиям, свежее сверху | Быстро вспомнить «что и когда меняли» |
| `deploy.md` | Пошаговая инструкция деплоя на чистый Cloudflare + Google Sheets | Развернуть проект с нуля |
| `specs/` | **Не в git.** Постановки текущей работы (внутри реальные суммы кредитов) | Что делаем прямо сейчас |
| `archive/` | Старые планы: реализованные **или** отложенные | Референс «как задумывали» |
| `private/` | **Не в git.** Реальные балансы, долги, id счетов, живой хэндофф, операционка аггрегатора | Личное закулисье — см. `private/README.md` |

## Правило деления: публичное vs приватное

Это open-source репо (`Kravchenko-Daniil/quick-expenses`). В публичный git идут
только `CHANGELOG.md`, `deploy.md`, `archive/` и этот README. **`docs/specs/` и
`docs/private/` — в `.gitignore`**, потому что содержат реальные финансовые данные
Даниила (суммы кредитов, долги, адреса кошельков). Любой документ с реальными
цифрами кладётся только туда.

## Поток работы

Текущая постановка живёт в `specs/`. Реализовали — запись о готовности в
`CHANGELOG.md`, саму постановку удаляем (сделанное не храним) либо, если нужна как
референс без реальных цифр, переносим в `archive/`.
