# docs/ — вся документация проекта

Единое место для всех документов. Код — в `api/`, `web/`, `scripts/`; всё, что
про проект словами (планы, инструкции, история, заметки) — здесь.

## Что в какой папке

| Путь | Что лежит | Зачем |
|------|-----------|-------|
| `CHANGELOG.md` | История изменений по сессиям, свежее сверху | Быстро вспомнить «что и когда меняли» |
| `deploy.md` | Пошаговая инструкция деплоя на чистый Cloudflare + Google Sheets | Развернуть проект с нуля |
| `specs/` | Планы **ещё не закрытых** фич | Что делаем / собираемся делать |
| `archive/` | Планы **уже реализованных** фич | Референс «как задумывали», когда фича готова |
| `private/` | **Не в git.** Реальные балансы, долги, id счетов, живой хэндофф | Личное закулисье — см. `private/README.md` |

## Правило деления: публичное vs приватное

Это open-source репо (`Kravchenko-Daniil/quick-expenses`). Всё в `docs/`, **кроме
`docs/private/`**, коммитится в публичный git. `docs/private/` — в `.gitignore`,
потому что содержит реальные финансовые данные Даниила. Любой документ с реальными
суммами/долгами/адресами кошельков кладётся только в `private/`.

## Когда фича готова

План из `specs/` переезжает в `archive/` (`git mv`), а запись о готовности — в
`CHANGELOG.md`. Так `specs/` всегда показывает только незавершённое.
