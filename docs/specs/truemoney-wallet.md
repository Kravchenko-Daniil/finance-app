# План: TrueMoney Wallet как отдельный THB-счёт

**Дата:** 2026-07-12 · **Статус:** account-token сделан (в git); авто-захват — research done, ждёт полевого теста владельца

## ⚠️ РАЗВОРОТ ЗАМЫСЛА (сессия 2026-07-12, после первого куска)

Изначально план был «TrueMoney = ручной дефолтный THB-счёт». Даниил уточнил настоящий
замысел: **TrueMoney читается АВТОМАТИЧЕСКИ** (он платит кошельком → траты сами влетают в
лог, как другие источники авто-аггрегатора). **Ручной ввод остаётся только для налички**
(`нал` → `cash`); наличкой он почти не пользуется.

**Следствие (важное):** переключать `DEFAULT_ACCOUNT_THB` на `truemoney` **НЕЛЬЗЯ** — если
поллер сам пишет траты TrueMoney, а ручной дефолт тоже кидает на truemoney, покупка
задвоится (капкан правил #4 `log_only`/snapshot). Поэтому:
- ✅ **account-token (`нал`→cash) — сделан и закоммичен** (`5ffdfdb`): ручная наличная тропа.
- ❌ **флип `DEFAULT_ACCOUNT_THB`→truemoney — отменён** (задвоил бы с авто-захватом).
- `cash` пока остаётся дефолтом THB. Токен `нал` станет нагруженным, когда truemoney начнёт
  питаться автоматом.

Главная фича теперь = **авто-захват транзакций TrueMoney** (новый источник аггрегатора,
см. `docs/private/aggregator-design.md`). Research по каналам — ниже.

## Зачем (исходная постановка, частично в силе)

Наличка (`cash`) остаётся для редких наличных платежей. TrueMoney — основной способ оплаты,
но питается автоматически, а не ручным вводом.

**Решения владельца (сессия 2026-07-12):**
- TrueMoney и `cash` **сосуществуют** (не замена).
- TrueMoney читается **автоматически** (поллер/листенер), ручное = только наличка.
- Пополнение TrueMoney = **transfer-событие** (банк/USDT/нал → truemoney). Механизм уже есть.

## Research: каналы авто-захвата TrueMoney (deep-research, 2026-07-12)

22 подтверждённых утверждения / 3 убито состязательной проверкой. Полный прогон:
`.../workflows/wf_b47dc3bc-d6a`. Вердикт по каналам:

- **Официальный API — тупик.** Всё официальное (2C2P PGW, Checkout.com/Alipay+, e-payment PHP) —
  merchant-приём платежей (списать с клиента), НЕ чтение своего кошелька. Personal read
  endpoint отсутствует.
- **Агрегаторы — тупик.** ZenMoney/Money Lover TrueMoney не поддерживают (ZenMoney — только
  RU/UA). Off-the-shelf нет.
- **Экспорт выписки — не найден.** Ни CSV/PDF/statement с автоматизацией.
- **① Push телефона → Telegram/webhook — рекомендуемый MVP.** `NotificationListenerService` —
  штатный Android-механизм, форвардеры пуш→HTTP рабочие (ZenMoney так и делает). Ложится на
  существующий MaxSwap-листенер. Низкий бан-риск (свой телефон, без имперсонации). Готового
  TrueMoney-парсера НЕТ — писать самому. **Не подтверждено**, шлёт ли TrueMoney пуш с суммой на
  каждую трату — только полевой тест с телефона.
- **② SMS-форвардер — фолбэк.** Паттерн SMS→HTTP рабочий, банковские тайские SMS парсятся; что
  TrueMoney шлёт парсибельный SMS — НЕ подтверждено (убито). Android-only.
- **③ Reverse-engineered API** (`pyTrueWallet`, Go `TrueWallet-API`) — технически читают личную
  историю+баланс (исходящие тоже) через `mobile-api-gateway.truemoney.com`. НО: высокий бан-риск
  (имперсонация, hardcoded HMAC), логин email+пароль+**SMS OTP** (не headless), код 2019–2020
  (~6 лет, вероятно сломан). Крайний резерв, не MVP.
- **Убито:** «официальный webhook TrueMoney» (нет — community, и про *приём* P2P); «`transaction-parser-th`
  поддерживает TrueMoney SMS» (нет — банковские SMS).

**Рекомендация:** канал ① (push → Telegram → парсер) первым. Reverse-API — резерв.

## Развилка, которую это создаёт

Роутинг трат сейчас — **по валюте, не по счёту** (`defaultAccountByCurrency`, `api/src/index.js:132`).
Два THB-счёта (`cash` + `truemoney`) по одной валюте не различить. Значит нужен
**account-token** — ключевое слово в тексте траты, выбирающее счёт на лету.

Дизайн: симметрично существующему currency-token.
- Дефолт для THB → `truemoney` (меняем `DEFAULT_ACCOUNT_THB`).
- Явный токен `нал/cash/кэш` → счёт `cash`.

```
«кофе 350 бат»      → truemoney   (дефолт THB)
«такси 80 нал»      → cash        (account-token)
«кофе 350»          → primary_account (без изменений)
«налог 500»         → primary_account (НЕ cash — граница слова защищает)
```

## Изменения кода (обратимо, тестируемо)

### 1. `api/src/index.js` — parseExpense + роутинг

- Новый `ACCOUNT_TOKEN_RE` с той же unicode-границей слова, что у `CURRENCY_TOKEN_RE`
  (`api/src/index.js:122`) — защита от `налог`/подстрок:
  ```js
  const ACCOUNT_TOKEN_RE = /(?<![\p{L}\p{N}_])(нал|наличка|наличкой|наличные|наличными|cash|кэш)(?![\p{L}\p{N}_])/giu;
  const TOKEN_ACCOUNT = { нал:'cash', наличка:'cash', наличкой:'cash', наличные:'cash', наличными:'cash', cash:'cash', кэш:'cash' };
  ```
- `parseExpense` возвращает доп. поле `account` (alias-ключ, напр. `'cash'`, или `null`).
  Тот же guard «ровно один токен» (`tokens.length === 1`), что у валюты; токен вырезается
  из текста до парсинга суммы.
- `accountAliasById(env)` — симметрично `defaultAccountByCurrency`: `{ cash: env.ACCOUNT_ALIAS_CASH }`.
  Alias → реальный id счёта через env (id не хардкодим).
- `handleQuickExpense` (`api/src/index.js:547`) — приоритет резолва `from`:
  1. `parsed.account` есть → `accountAliasById(env)[parsed.account]` (самый специфичный, побеждает валюту)
  2. иначе `parsed.currency` → currency-default
  3. иначе → `primary_account`

  Неизвестный alias без env-конфига → `error(500, ...)` в стиле существующих проверок.

### 2. `api/test-smoke.mjs` — синхрон инлайн-копии + кейсы

Инлайн-копия `parseExpense`/токенов в тестах должна повторить прод (CLAUDE.md #5). Добавить кейсы:
- `«такси 80 нал»` → `{account:'cash', amount:80, currency:null}`
- `«кофе 350»` → `account:null`
- **`«налог 500»` → `account:null`** (критичный guard: подстрока не ловится)
- `«обед 200 бат нал»` → `currency:'THB'` + `account:'cash'` (оба токена)
- `«нал 100 cash»` (два account-токена) → `account:null` (зеркалит guard «ровно один»)
- `«кофе 350 cash»` → `account:'cash'`
- существующие валютные кейсы — без регресса.

**Критерий готовности:** `cd api && node test-smoke.mjs` — все зелёные, exit 0.

## Изменения конфига (прод) — ПЕРЕСМОТРЕНЫ после разворота

- ❌ **`DEFAULT_ACCOUNT_THB = "truemoney"` — ОТМЕНЕНО.** Задвоило бы с авто-захватом. `cash`
  остаётся дефолтом THB, пока авто-источник TrueMoney не поднят.

  500 — но он пока и не нужен (наличка = текущий дефолт THB).
- primary_account не трогаем.
